# Claude Token Tracker

Track and **reduce** your Claude Code token usage and estimated costs. Parses conversation transcripts from `~/.claude/projects/`, provides a CLI reporter and web dashboard for analytics, and includes an output compressor that intercepts verbose CLI output before it reaches the LLM context — cutting token waste automatically.

Zero npm dependencies. Pure Node.js.

## Token Tracker CLI

The tracker scans `~/.claude/projects/` for `.jsonl` transcript files, extracts token usage from each assistant response, and reports aggregated costs.

### Basic Usage

```bash
node src/tracker/index.js              # All-time summary
node src/tracker/index.js --today      # Today only
node src/tracker/index.js --yesterday  # Yesterday only
node src/tracker/index.js --week       # This week
node src/tracker/index.js --month      # This month
```

### Time Filters

```bash
node src/tracker/index.js --days 7                           # Last 7 days
node src/tracker/index.js --from 2026-03-01 --to 2026-03-25 # Custom range
```

### Grouping

```bash
node src/tracker/index.js --by-project    # Break down by project
node src/tracker/index.js --by-model      # Break down by model
node src/tracker/index.js --daily         # Day-by-day breakdown
```

Combine freely:

```bash
node src/tracker/index.js --week --by-project --by-model
node src/tracker/index.js --days 30 --daily
```

### Export Formats

```bash
node src/tracker/index.js --month --csv    # CSV output
node src/tracker/index.js --month --json   # JSON output
```

### What It Tracks

| Metric | Description |
|--------|-------------|
| Input tokens | Tokens sent to Claude |
| Output tokens | Tokens generated by Claude |
| Cache read tokens | Tokens served from prompt cache |
| Cache creation tokens | Tokens used to populate prompt cache |
| Estimated cost | Calculated from per-model pricing rates |
| Sessions | Unique conversation sessions |
| API calls | Individual model requests |

## Web Dashboard

A browser-based dashboard for visualizing token usage, costs, and compression savings.

```bash
node src/dashboard-claude/server.js              # Start on default port 3456
node src/dashboard-claude/server.js --port 8080  # Custom port
```

Open `http://localhost:3456` in your browser.

### Dashboard Features

- **Summary cards** — total estimated cost, token counts, cache tokens, sessions, API calls
- **Daily cost chart** — bar chart of spending over time
- **Model breakdown** — doughnut chart showing cost distribution across Claude models
- **Token breakdown** — doughnut chart of input vs output vs cache read vs cache creation
- **Activity heatmap** — cost by hour-of-day and day-of-week, showing usage patterns
- **Projects table** — all projects ranked by cost with inline trend bars
- **Daily breakdown table** — per-day token counts and costs
- **Compression stats panel** — total savings, per-command breakdown, recent compression runs
- **Date range filters** — Today, Yesterday, This Week, This Month, All Time, or custom range
- **Refresh button** — reload data without restarting the server

### API Endpoints

| Endpoint | Description |
|----------|-------------|
| `GET /` | Serve the dashboard HTML |
| `GET /api/data?from=YYYY-MM-DD&to=YYYY-MM-DD` | Token usage data (summary, daily, projects, models, heatmap) |
| `GET /api/compress-stats?from=&to=` | Compression statistics (summary, daily savings, recent runs) |
| `GET /api/refresh` | Clear server-side cache and reload data |

Data is cached server-side with a 1-minute TTL.

## Output Compressor

The compressor reduces verbose CLI output before it enters Claude's context window. It strips noise (ANSI codes, spinners, boilerplate hints) while preserving actionable information.

### Pipe Usage

```bash
git status | node bin/compress.js git status
npm test   | node bin/compress.js npm test
docker ps  | node bin/compress.js docker ps
dotnet build | node bin/compress.js dotnet build
```

The argument after `bin/compress.js` tells the compressor which command produced the output, so it can apply the right rule set.

### Compression Filters

These core filters are applied to all output:

| Filter | What It Does |
|--------|-------------|
| **stripAnsi** | Removes ANSI escape codes, color sequences |
| **stripProgress** | Removes spinners, progress bars, percentage indicators |
| **stripHints** | Drops boilerplate hints (git advice, npm warnings) |
| **collapseBlankLines** | Collapses consecutive blank lines to a single one |
| **dedup** | Collapses repeated identical lines → `[xN] line content` |
| **groupSimilar** | Aggregates similar lines (e.g., `modified: file1, file2 (+3 more)`) |
| **truncate** | Caps long output at max lines, keeping head + tail with a summary |

### Command-Specific Rules

On top of the core filters, specialized rules parse and compress output from specific tools:

| Rule Set | Commands | Highlights |
|----------|----------|------------|
| **git** | `git status`, `git log`, `git diff`, `git push/pull/fetch` | Parses status into compact branch/staged/modified/untracked summary |
| **npm** | `npm install`, `npm test`, `npm run build`, yarn, pnpm | Extracts package counts, test pass/fail results, build status |
| **docker** | `docker build`, `docker ps`, `docker logs`, `docker pull/push` | Summarizes build steps, truncates tables, deduplicates log lines |
| **dotnet** | `dotnet build`, `dotnet test`, `dotnet ef`, `dotnet restore` | Extracts build success/errors/warnings, test counts, migration lists |
| **general** | tsc, eslint, prettier, cargo, kubectl, terraform, curl, and more | Configurable max-line limits per tool category |

## Shell Aliases

Instead of manually piping each command, source the aliases file to auto-compress all supported commands:

```bash
source shell/aliases.sh
```

Add this to your `~/.bashrc` for persistent activation:

```bash
# Auto-compress CLI output for Claude Code
source /path/to/token-tracker/shell/aliases.sh
```

### Control Commands

| Command | Description |
|---------|-------------|
| `tt_off` | Disable compression (unalias all wrapped commands) |
| `tt_on` | Re-enable compression (re-source the aliases) |
| `tt_raw <cmd>` | Bypass compression for a single command |

## Compression Statistics

Every compression run is logged to `~/.claude/token-tracker/compress-log.jsonl`. Use the stats viewer to see your savings.

```bash
node bin/stats.js              # All-time summary
node bin/stats.js --today      # Today's stats only
node bin/stats.js --days 7     # Last 7 days
node bin/stats.js --tail 20    # Last 20 individual compression runs
node bin/stats.js --raw        # Raw JSON output
node bin/stats.js --clear      # Clear the log file
```

The stats viewer shows:
- **Total compression runs** and total tokens saved
- **By-command breakdown** — which commands produce the most savings
- **Recent runs** — individual compression events with before/after token counts

Token estimation: ~4 characters per token.

The web dashboard also displays compression stats in a dedicated panel.

## How It Works

### Token Tracking

Claude Code stores conversation transcripts as `.jsonl` files in `~/.claude/projects/`. Each assistant response includes a `usage` object with token counts (input, output, cache read, cache creation). The tracker scans those files, filters by date/project, aggregates the data, and calculates costs using per-model pricing rates.

All data is **read-only** — nothing in `~/.claude/` is modified.

### Output Compression

The compressor sits between CLI tool output and Claude's context:

```
Claude runs "git status"
  → shell alias intercepts the command
  → runs the real command
  → pipes stdout+stderr through node bin/compress.js
  → compressor applies filters + command-specific rules
  → Claude receives compressed output (fewer tokens)
```

### Data Storage

No database. All data is file-based:

| File | Purpose |
|------|---------|
| `~/.claude/projects/**/*.jsonl` | Claude Code transcripts (read-only) |
| `~/.claude/token-tracker/compress-log.jsonl` | Compression run logs (append-only) |

### Authentication

None. Everything runs locally. The dashboard binds to localhost only. No external network calls.
